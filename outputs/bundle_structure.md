# Bundle Structure v2

## Customer-Facing File Map (v2 Manifest)

### Core Docs

1. `start_here.md`
   - First-read onboarding document with 30-minute first-value flow.

2. `scorecard.md`
   - 40-question weighted readiness diagnostic with revenue-readiness output.

3. `teardown.md`
   - Composite launch case study with launch-day decisions and Day 7/30 revenue follow-through.

4. `playbook_v3.1.md`
   - Operating manual with countdown, launch-day cadence, and post-launch revenue SOP.

5. `quick_reference_card.md`
   - One-page top-10 action summary for rapid execution.

### Implementation Assets

6. `revenue_command_center.csv`
   - Daily metric command center from sessions to paid revenue.

7. `launch_cohort_tracker.csv`
   - User-level launch cohort lifecycle tracker.

8. `activation_sequence_templates.md`
   - Day 0/1/3/7 segment templates for email and in-app messaging.

9. `offer_experiment_backlog.csv`
   - Offer and message test backlog with outcome decisions.

10. `evidence_register.md`
   - Claim, evidence basis, and confidence register.

## Packaging Notes

- Keep filenames stable to reduce support overhead.
- Do not include internal build docs in the customer ZIP.
- Export to PDF only after final proofread.
- For future updates, append semantic suffixes (for example, `playbook_v3.3.md`) and preserve prior versions for existing buyers.
