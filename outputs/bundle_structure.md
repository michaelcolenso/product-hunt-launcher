# Bundle Structure

## Customer-Facing File Map

1. `start_here.md`
   - First-read onboarding document.
   - Explains sequence, expectations, and immediate next action.

2. `scorecard.md`
   - 40-question weighted readiness diagnostic.
   - Produces category-level gaps before launch scheduling.

3. `teardown.md`
   - Full first-person composite case study.
   - Shows real launch-day decisions, numbers, and tradeoffs.

4. `playbook_v3.1.md`
   - Operational execution manual.
   - Includes countdown phases, asset checklists, swipe files, rescue protocol, and appendices.

5. `quick_reference_card.md`
   - One-page top-10 action summary for rapid execution.

## Internal Build Assets

6. `landing_page_brief.md`
   - Copy and strategy brief for web implementation.

7. `gumroad_setup.md`
   - Listing configuration, pricing, upload checklist, and receipt email copy.

## Recommended Delivery Order in Product Download

1. `start_here.md`
2. `scorecard.md`
3. `teardown.md`
4. `playbook_v3.1.md`
5. `quick_reference_card.md`

## Packaging Notes

- Keep filenames stable to avoid support confusion.
- Use markdown exports to PDF only after final proofreading pass.
- If you add versions later, append semantic suffixes (for example, `playbook_v3.2.md`) and preserve prior files for existing buyers.
